# Plan 01-02: Transport & FastAPI Integration

**Wave 2 of Phase 1** — Dual transport support (stdio + HTTP), FastAPI mounting, metatron_sync tool.

## Tasks

### Task 1.6: Add metatron_sync Tool
**Purpose:** SYNC-01 - Trigger document sync from configured sources

Read existing `src/metatron/mcp/sync.py` to understand current implementation, then modify `tools.py`:

Add `metatron_sync` tool:
- Params: source?, workspace_id?, force_full?
- Description: "Sync documents from configured sources..."
- Delegates to: existing MCP sync manager or create new `MCPSyncManager`
- Returns: success, sync_result with sources_synced, details[]

**Verify:** Tool appears in mcp._tools list

### Task 1.7: Create HTTP Auth Middleware
**Purpose:** API key validation for HTTP transport

```bash
touch src/metatron/mcp/auth.py
```

Create `src/metatron/mcp/auth.py`:
- `validate_api_key(authorization_header) -> bool`
- Check for `Authorization: Bearer <key>` header
- Validate against `METATRON_MCP_API_KEY` env var
- Allow all if no key configured (dev mode)

**Verify:** `python -c "from metatron.mcp.auth import validate_api_key; import os; os.environ['METATRON_MCP_API_KEY']='test'; print(validate_api_key('Bearer test'))"`

### Task 1.8: Configure Dual Transport in Server
**Purpose:** TRNS-01 (stdio) + TRNS-02 (StreamableHTTP)

Modify `src/metatron/mcp/server.py`:
- Add transport selection based on environment
- For stdio: read config from ~/.metatron/config.json
- For HTTP: enable stateless_http=True, require API key auth
- Add `--transport`, `--host`, `--port` CLI arguments

**Verify:** Server runs with both transports

### Task 1.9: Create Server Entry Point
**Purpose:** Executable entry point for stdio and HTTP modes

```bash
touch src/metatron/mcp/__main__.py
```

Create `src/metatron/mcp/__main__.py`:
- CLI argument parsing (transport, host, port)
- Configure logging to stderr BEFORE any MCP code (CRITICAL)
- Import tools module to register all tools
- Run with appropriate transport

**Verify:** `python -m metatron.mcp --transport stdio` starts without errors

### Task 1.10: Mount MCP to Existing FastAPI App
**Purpose:** TRNS-03 - Shared lifespan FastAPI integration

Read existing `src/metatron/api/app.py`, then modify:

```python
from metatron.mcp.server import mcp

# Create MCP HTTP app
mcp_app = mcp.http_app(
    path="/mcp",
    transport="streamable-http",
    stateless_http=True,
)

# Mount with shared lifespan
api = FastAPI(lifespan=mcp_app.lifespan)  # CRITICAL
api.mount("/mcp", mcp_app)
```

Add `/mcp` health endpoint if needed.

**Verify:** GET /mcp returns MCP server info
