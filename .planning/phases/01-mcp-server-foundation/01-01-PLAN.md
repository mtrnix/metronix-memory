# Plan 01-01: MCP Server Core Tools

**Wave 1 of Phase 1** — Build the foundation: FastMCP server instance, tool implementations with proper descriptions, structured error handling, cursor-based pagination.

## Tasks

### Task 1.1: Create FastMCP Server Instance
**Purpose:** Establish the MCP server container with proper logging configuration

```bash
touch src/metatron/mcp/server.py
```

Create `src/metatron/mcp/server.py`:
- Initialize FastMCP server named "MetatronMCP"
- Configure structlog to stderr (CRITICAL: stdout is for JSON-RPC)
- Set server instructions for LLM tool selection
- Version: 1.0.0

**Verify:** Server can be imported without errors

### Task 1.2: Create Structured Error System
**Purpose:** Consistent error format across all tools (code, message, hint)

```bash
touch src/metatron/mcp/errors.py
```

Create `src/metatron/mcp/errors.py`:
- `MCPError` model with code, message, hint, details fields
- Error code constants: WORKSPACE_NOT_FOUND, QDRANT_UNAVAILABLE, MEMGRAPH_UNAVAILABLE, INVALID_CURSOR, INTERNAL_ERROR
- `handle_tool_error(tool_name, exception) -> MCPError` function

**Verify:** `python -c "from metatron.mcp.errors import MCPError, handle_tool_error"`

### Task 1.3: Create Cursor-Based Pagination Helpers
**Purpose:** Stable pagination for search results

```bash
touch src/metatron/mcp/pagination.py
```

Create `src/metatron/mcp/pagination.py`:
- `encode_cursor(data) -> str` - base64 encode JSON
- `decode_cursor(cursor) -> dict` - decode with error handling
- `CursorPager` class with paginate(results, cursor) -> (items, has_more, next_cursor)

**Verify:** `python -c "from metatron.mcp.pagination import CursorPager; p = CursorPager(limit=2); print(p.paginate([1,2,3,4], None))"`

### Task 1.4: Implement Tool Functions (search, get, store, status)
**Purpose:** Core MCP tools with proper descriptions for LLM selection

```bash
touch src/metatron/mcp/tools.py
```

Create `src/metatron/mcp/tools.py` with these tools:

1. **metatron_search** (MCP-01, MCP-06)
   - Params: query, workspace_id?, limit (1-100), cursor?, include_graph?
   - Description: "Search the knowledge base using hybrid RAG..." (see RESEARCH.md for full text)
   - Delegates to: `retrieval.search.hybrid_search_and_answer` (or lower-level for structured results)
   - Returns: SearchResponse with results[], has_more, next_cursor, total

2. **metatron_get** (MCP-02)
   - Params: doc_label, workspace_id?
   - Description: "Retrieve a specific document by label..."
   - Delegates to: `QdrantVectorStore.search_by_doc_labels()`
   - Returns: Document with content, title, timestamp, source_type

3. **metatron_store** (MCP-03)
   - Params: content, title?, workspace_id?, doc_label?, metadata?
   - Description: "Store a new document or memory..."
   - Delegates to: `ingestion.pipeline.ingest_documents()`
   - Returns: success, doc_label, chunks_stored

4. **metatron_status** (MCP-04)
   - Params: workspace_id?
   - Description: "Check system health and status..."
   - Returns: status, documents (total, by_source), last_sync, embedding_model

**Verify:** `python -c "from metatron.mcp.tools import mcp; print([t.name for t in mcp._tools])"`

### Task 1.5: Update Stdio Config Loader
**Purpose:** Load workspace context from ~/.metatron/config.json for stdio transport

Read existing `src/metatron/mcp/config.py` and enhance:
- CONFIG_PATH = Path.home() / ".metatron" / "config.json"
- Load on import, provide default workspace_id
- Expose: default_workspace_id, api_key properties

**Verify:** Config file at ~/.metatron/config.json loads correctly
