  <files>docker-compose.yml</files>
  <action>
    Create production docker-compose.yml with 4 services and proper dependency ordering:

    **PostgreSQL Service:**
    - Image: postgres:16-alpine
    - Port: 5432 (no host binding, Docker network only)
    - Environment:
      - POSTGRES_USER=metatron
      - POSTGRES_PASSWORD=metatron_dev (development only - note for security)
      - POSTGRES_DB=metatron

    - Health check: `pg_isready -U metatron` every 5s, 30s timeout, 5 retries
    - Networks: metatron_network

    **Qdrant Service:**
    - Image: qdrant/qdrant:latest
    - Port: 6333 (no host binding)
    - Volume: qdrant_data:/qdrant/storage
    - Environment: QDRANT_API_KEY=qdrant_dev (development only)
    - Health check: `curl -f http://localhost:6333/health` every 5s, 30s timeout, 5 retries
    - Networks: metatron_network

    **Memgraph Service:**
    - Image: memgraph/memgraph:latest
    - Port: 7687 (Bolt protocol, no host binding)
    - Port: 7444 (web UI - expose to localhost:7444 for debugging)
    - Volume: memgraph_data:/var/lib/memgraph
    - Environment: MG_BOLT_PORT=7687
    - Health check: `timeout 3 nc -w 1 localhost 7687 </dev/null` every 5s, 30s timeout, 5 retries
    - Networks: metatron_network

    **Metatron Service:**
    - Build: ./docker/Dockerfile.metatron
    - Port: 8000:8000 (expose to localhost:8000 for API access)
    - Environment:
      - DATABASE_URL=postgresql://metatron:metatron_dev@postgres:5432/metatron
      - QDRANT_HOST=qdrant
      - QDRANT_PORT=6333
      - MEMGRAPH_HOST=memgraph
      - MEMGRAPH_PORT=7687
      - METATRON_MCP_API_KEY=dev_key_for_testing (development only)
      - PYTHONUNBUFFERED=1
    - Depends on: postgres, qdrant, memgraph (all with `condition: service_healthy`)
    - Health check: `curl -f http://localhost:8000/health` every 10s, 60s timeout, 5 retries
    - Networks: metatron_network
    - Restart policy: unless-stopped

    **Named Volumes:** postgres_data, qdrant_data, memgraph_data
    **Networks:** metatron_network (bridge mode)

    Comment in the file:
    ```

    # For production, use docker-compose.prod.yml with secrets management.
    ```
