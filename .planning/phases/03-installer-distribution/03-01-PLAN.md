<task type="auto">
  <name>Task 1: Create install.sh with dependency checking and setup automation</name>
  <files>install.sh</files>
  <action>
Create executable shell script `install.sh` (bash 4+) that:

1. **Header & Safety:**
   - Shebang: `#!/bin/bash`
   - Set `set -euo pipefail` for error handling
   - Add usage comment explaining one-line usage

2. **Color codes for output** (for clear feedback):
   - RED='\033[0;31m', GREEN='\033[0;32m', YELLOW='\033[1;33m', NC='\033[0m'

3. **Python 3.12+ Detection:**
   - Try `python3 --version` OR `python --version`
   - Parse version string: extract major.minor (e.g., "3.12")
   - Compare: if < 3.12, print error in RED and exit 1
   - If found, print GREEN checkmark: "✓ Python 3.12+ detected (X.Y.Z)"

4. **Docker Detection:**
   - Try `docker --version`
   - If found, print GREEN checkmark: "✓ Docker detected"
   - If NOT found, print YELLOW warning: "⚠ Docker not found. Installation requires Docker. Install from https://docs.docker.com/get-docker/"
   - Continue anyway (Docker is optional in terms of detection, but required for setup later)

5. **Git Check:**
   - Try `git --version`
   - If NOT found, print error in RED and exit 1
   - Print GREEN checkmark if found

6. **Clone Repository:**
   - Prompt user: "Enter GitHub URL or press Enter for default (https://github.com/openclaw/metatron.git)"
   - Default to: https://github.com/openclaw/metatron.git
   - Create temp dir: `INSTALL_DIR="${HOME}/.metatron"` (idempotent: remove if exists)
   - Clone with timeout: `git clone --depth 1 "$REPO_URL" "$INSTALL_DIR"`
   - If clone fails, print error and exit 1

7. **Setup Docker Compose Stack:**
   - Change to install directory: `cd "$INSTALL_DIR"`
   - Check docker-compose.yml exists (error if not)
   - Run: `docker-compose pull` (download images)
   - Run: `docker-compose up -d` (start services)
   - If docker-compose fails, print error and exit 1

8. **Success Message:**
   - Print GREEN message: "✓ Metatron installed successfully!"
   - Print next steps:
     ```
     Next steps:
     1. Wait for services to be healthy: docker-compose logs -f metatron
     2. Open http://localhost:8000 in your browser
     3. See docs/QUICKSTART.md for first steps
     ```

9. **Error Handling:**
   - All commands wrapped with error checking: `|| { echo "ERROR: ..."; exit 1; }`
   - No bare `set -e` reliance — explicit error checks
   - Print helpful context on each failure (what command, why it matters)

**Security Requirements (INST-03 — must be documented in comments):**
- ✓ No eval() anywhere
- ✓ All user input quoted: `"$REPO_URL"` (not $REPO_URL)
- ✓ No downloading/executing scripts; only git clone then run existing docker-compose.yml
- ✓ Script itself is served via HTTPS (documented in INSTALL.md)
- ✓ Checksum verification documented (user can verify before piping to bash)

Make script idempotent — safe to run multiple times without failure. If $HOME/.metatron exists, warn user and ask to proceed (rm -rf or use existing).
  </action>
