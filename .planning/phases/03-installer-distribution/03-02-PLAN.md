1. **Create docs/DISTRIBUTION.md:**
   - **Title:** "Metatron Installer Distribution Guide"
   - **Overview:** Explains how installer is distributed to users securely
   - **Section: GitHub Releases (Recommended):**
     - Instructions for downloading from GitHub releases page
     - URL pattern: `https://github.com/openclaw/metatron/releases/download/v{VERSION}/install.sh`
     - Note: Checksum available as `install.sha256` in same release
   - **Section: Raw GitHub (Alternative):**
     - Direct URL: `https://raw.githubusercontent.com/openclaw/metatron/main/install.sh`
     - Warning: Always use with checksum verification (see INSTALL.md)
   - **Section: Custom CDN/Static Site:**
     - Instructions for deploying to custom domain
     - Example: `https://app.mtrnix.com/install.sh`
     - Setup with nginx/Cloudflare/similar
   - **Security Notes:**
     - Explain HTTPS requirement
     - Link to checksum verification (INSTALL.md)
     - Recommend users verify before piping to bash
   - **Maintenance:**
     - How to update installer when new version is released
     - GitHub Actions workflow runs automatically on new release
   - **Troubleshooting:**
     - 404 errors when downloading
     - Checksum mismatch reasons

2. **Create .github/workflows/release-installer.yml:**
   - **Trigger:** On GitHub release creation (types: [published])
   - **Jobs:**
     - Checkout code
     - Verify install.sh syntax: `bash -n install.sh`
     - Generate checksum: `sha256sum install.sh > install.sha256`
     - Upload installer as release asset: `install.sh`
     - Upload checksum as release asset: `install.sha256`
   - **Example workflow structure:**
     ```yaml
     name: Release Installer
     on:
       release:
         types: [published]
     jobs:
       release:
         runs-on: ubuntu-latest
         steps:
           - uses: actions/checkout@v4
           - name: Verify installer syntax
             run: bash -n install.sh
           - name: Generate checksum
             run: |
               sha256sum install.sh > install.sha256
               cat install.sha256
           - name: Upload assets to release
             uses: softprops/action-gh-release@v1
             with:
               files: |
                 install.sh
                 install.sha256
     ```
   - **Note:** Assets become available at: `https://github.com/openclaw/metatron/releases/download/{TAG}/install.sh`

3. **Update Makefile** with targets:
   - `make test-installer` — Run install.sh syntax check: `bash -n install.sh`
   - `make verify-checksum` — Verify checksum: `sha256sum -c .sha256sum`
   - `make update-checksum` — Regenerate: `sha256sum install.sh > .sha256sum`
   - `make prepare-release` — Runs tests + checksum verification (used before creating release)
   - Example:
     ```makefile
     test-installer:
     	bash -n install.sh
     
     verify-checksum:
     	sha256sum -c .sha256sum
     
     update-checksum:
     	sha256sum install.sh > .sha256sum
     
     prepare-release: test-installer verify-checksum
     	@echo "✓ Installer ready for release"
     ```
